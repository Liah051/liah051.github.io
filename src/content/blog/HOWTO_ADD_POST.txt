■ 記事（ブログポスト）の作成手順

新しい記事を作成する場合、`src/content/blog/` 内にMarkdown（`.md`）またはMDX（`.mdx`）ファイルを作成します。
ブログを整理しやすくするために、関連するカテゴリのフォルダ（例：`physics/`, `programming/` など）を作って、その中にファイルを配置することも可能です。

■ フロントマター（Frontmatter）の書き方
ファイルの先頭には `---` で囲まれたYAML形式のメタデータ（フロントマター）を記述します。

【最低限必要な項目】
```yaml
---
title: 記事のタイトル
pubDatetime: 2026-04-07T00:00:00Z
description: 記事の概要や説明文。検索結果やSNSのプレビューなどで表示されます。
---
```

【よく使うオプション項目を含めた例】
```yaml
---
title: 物理学の基礎入門
pubDatetime: 2026-04-07T12:00:00+09:00 # 日本時間の場合は+09:00を付けると便利です
modDatetime: 2026-04-08T10:00:00+09:00 # 更新した日時（任意）
description: 物理学の基本的な考え方について解説します。
author: あなたの名前 # 省略した場合はサイトのデフォルト管理者が設定されます
featured: true # trueにするとトップページなどで「おすすめ」枠として強調表示されます
draft: false # trueにすると下書き扱いになり、本番環境には表示されません
tags:
  - 物理学
  - 勉強
category_id: physics # サイドバーを切り替えるためのID（MobileDrawerやMainLayoutで設定したもの）
group: physics-series # 続き物の記事（シリーズ）としてまとめたい場合のグループ名
order: 1 # シリーズ内での表示順序（数字が小さいほど先頭）
---
```

■ フロントマターの全項目（スキーマ）一覧
- `title` (必須): 記事のタイトル
- `pubDatetime` (必須): 公開日時（例：2026-04-07T14:30:00Z）
- `description` (必須): 記事の簡単な説明文
- `author` (任意): 著者名。デフォルトは設定ファイルの管理者の名前になります。
- `modDatetime` (任意): 記事の更新日時。修正した際に追記します。
- `featured` (任意): おすすめ記事として目立たせるか（true / false）
- `draft` (任意): 下書き状態にするか（true / false）。trueならローカルサーバーの開発モードでのみ確認できます。
- `tags` (任意): タグのリスト。配列（ハイフンリスト）で指定します。
- `category_id` (任意): 記事のカテゴリID。このIDに基づいて、PC用・スマホ用のサイドバーの表示が切り替わります（例：`physics`, `blue-protocol` など）。
- `group` (任意): シリーズものとして記事をまとめたい場合に設定する共通のグループ名。
- `order` (任意): シリーズもの（groupを指定した場合）の中での順番を決める数値。
- `ogImage` (任意): SNSでシェアされた時の画像URL、またはローカルの画像パス。
- `thumbnail` (任意): 記事一覧などで表示されるサムネイル画像のURL/パス。
- `canonicalURL` (任意): SEO用の正規URL。
- `hideEditPost` (任意): 記事の下の「編集」ボタンを隠すか（true / false）。
- `timezone` (任意): タイムゾーン文字列。
- `wordlist` (任意): 用語集（Glossary）リストを表示するか（true / false）。
- `terms` (任意): 記事に関連づけたい用語（Glossary）の配列。

■ MDXファイルについて
拡張子を `.mdx` にすると、記事内でAstroのコンポーネント（用語集の `<Word>` コンポーネントなど）を使うことができるようになります。特に理由がなければ `.md` をお使いいただけますが、リッチな表現をしたい場合は `.mdx` を使用してください。
