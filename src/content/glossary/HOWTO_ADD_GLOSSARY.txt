■ 用語（Glossary）の追加手順

ブログ内でツールチップや用語集として表示される用語（Glossary）の追加手順です。
`src/content/glossary` の中にカテゴリのフォルダを作り、そこにYAMLファイルを作成します。

1. フォルダの作成（任意）
   `src/content/glossary` の中に、用語に関連するカテゴリ名のフォルダを作成してください。
   （例：`src/content/glossary/physics/` や `src/content/glossary/blue-protocol/` など）
   既存のフォルダに用語を追加する場合は、そのまま進んでください。

2. YAMLファイルの作成
   追加したい用語の名前でYAMLファイルを作成します。
   （例：`加速度.yaml` や `重力.yaml` など）

3. 用語データ（スキーマ）の記入
   ファイルの中身には、以下の形式でデータを記入してください。

   ```yaml
   term: 用語の名前
   description: その用語についての説明文。なるべく簡潔に書くとツールチップで見やすくなります。
   url: https://example.com/ （※オプション：より詳細な説明がある外部リンクなどを指定）
   image: /assets/images/example.png （※オプション：関連する画像のパス）
   ```

   ※ `url` と `image` は省略可能（オプション）です。省略する場合はその行を書かなくて大丈夫です。

【項目（スキーマ）の詳細】
- term (必須): 用語の名前（文字列）。記事内の `<Word id="ファイル名">用語</Word>` と紐づく場合などに使われます。
- description (必須): 用語の説明文（文字列）。
- url (任意): 詳細ページのURL（文字列）。
- image (任意): 画像のパス（文字列）。

■ 記事内での使い方（参考）
記事（Markdown/MDX）の中で追加した用語を使うときは、大文字の `Word` タグを使います。
例：
```mdx
このように <Word>加速度</Word> を計算します。
```
※ もし「表示するテキスト」と「ファイル名（またはterm）」が違う場合は、以下のように `id` 属性でYAMLファイルの名前（拡張子なし）やtermを指定してください。
```mdx
このように <Word id="加速度">加速</Word> します。
```
